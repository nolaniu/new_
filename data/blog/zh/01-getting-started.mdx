欢迎来到学习工作室！

本项目参考 [timlrx/tailwind-nextjs-starter-blog](https://github.com/timlrx/tailwind-nextjs-starter-blog) 的结构，保留 `pages` 路由与 MDX 驱动的内容体系，同时根据需求加入效率工具页。

### 目标拆解

1. **博客体系**：使用 MDX 写作，存放在 `data/blog/*.mdx`。借助前言区块（frontmatter）维护标题、摘要、标签与阅读时长。
2. **站点结构**：`/` 为总览，`/blog` 展示所有学习记录，`/tools` 集中效率工具。
3. **视觉形象**：Tailwind CSS 的品牌色设定为紫色梯度，保持轻量与未来感。

### 配置 Tailwind 与 MDX

- Tailwind 在 `tailwind.config.js` 中声明扫描路径，并扩展字体与阴影。
- 通过 `next-mdx-remote` 在构建阶段序列化 MDX，搭配 `remark-gfm` 和 `rehype-autolink-headings` 获得更好的 Markdown 体验。

<Callout>
记得将常用组件（如 Callout、Tip、Note）收纳到 `components/` 中，MDX 可以直接引入 React 组件。
</Callout>
